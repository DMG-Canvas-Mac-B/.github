## 📖 Overview

<a href="#dMGCanvas">DMG Canvas</a> is the professional tool for creating polished DMG disk images for Mac application distribution — providing a visual canvas editor for designing the installer window appearance, then building the DMG file with code signing and notarization for modern macOS distribution requirements.

The <a href="#dMGCanvas">visual DMG layout editor</a> designs the disk image window appearance through direct manipulation — setting the window size, placing the application icon and Applications folder alias at exact positions, configuring the background image that gives the installer its visual character, and previewing the result exactly as users will see it when they mount the DMG. Professional Mac applications present their installers through carefully designed DMG windows that guide users through the drag-to-Applications installation action with clear visual communication. The <a href="#dMGCanvas">background image configuration</a> sets a custom background image for the DMG window — the visual design element that differentiates a polished professional installer from a plain Finder window. Background images can provide installation instructions, branding, arrow graphics indicating where to drag the application, and any visual design that communicates the installation action.

The <a href="#dMGCanvas">license agreement integration</a> embeds a license text file that displays to users before the DMG mounts — requiring users to accept the terms before accessing the disk image contents. This standard distribution practice for commercial software presents the End User License Agreement as part of the installation experience. The <a href="#dMGCanvas">code signing and notarization support</a> integrates the signing and notarization workflow that Apple requires for modern Mac application distribution — signing the DMG with a Developer ID certificate and notarizing with Apple's notarization service to pass Gatekeeper without warning dialogs on recipient Macs.

<a href="#dMGCanvas">Command-line interface for automation</a> builds DMG files from DMG Canvas project files through Terminal commands — integrating DMG creation into CI/CD pipelines, build scripts, and automated release workflows that run without a GUI. <a href="#dMGCanvas">Project file format</a> saves the complete DMG configuration as a file — window size, icon positions, background, license, signing settings — for reproducible DMG builds that produce identical results from the same project file across builds.

<a href="#dMGCanvas">Icon size and position control</a> sets the exact pixel positions and display sizes of icons in the DMG window — professional-looking icon arrangement with precise control over placement that looks deliberate rather than arbitrary. <a href="#dMGCanvas">Multiple DMG format options</a> builds compressed DMGs for smaller download sizes or uncompressed for faster mounting, with format selection appropriate for different distribution contexts.

## 💬 FAQ

<details>
<summary>What is a DMG file?</summary>

A DMG (Disk Image) is the standard distribution format for Mac applications — a virtual disk that mounts in Finder, typically containing the application and an Applications folder alias for drag-to-install.

</details>

<details>
<summary>Can DMG Canvas create custom background images for the installer?</summary>

Yes. A custom background image configures for the DMG window — providing branding, installation instructions, and visual design for the installer.

</details>

<details>
<summary>Does DMG Canvas support code signing?</summary>

Yes. Developer ID certificate signing integrates into the DMG build process.

</details>

<details>

Yes. Notarization submission to Apple's notarization service integrates into the build workflow for Gatekeeper compliance.
